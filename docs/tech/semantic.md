##1. Основные понятия    

###1.1 Алгоритм и блок-схема        
1.1.alg  Алгоритм — точно описанная последовательность действий, ведущая к поставленной цели.    
1.1.valg  Визуальный алгоритм — алгоритм, изображенный не в виде текста, а в виде наглядной картинки.(1.1.alg)    
1.1.prog   Программа — последовательность действий, которые человек ленится выполнять сам и поэтому поручает компьютеру или роботу.    
1.1.block  Блок-схема - основной вариант отображения визуального алгоритма.(1.1.valg)    
1.1.bread  Чтение блок-схемы производят сверху-вниз по стрелкам.(1.1.block)    
1.1.bfig   Блок-схема состоит из фигур.(1.1.block)    
1.1.blow   Чем ниже фигура в блок-схеме, тем позже происходит её выполнение.(1.1.block)    
1.1.oval   Обрамления в блок-схеме обозначаются овалом.(1.1.block)    
1.1.obr    Обрамления в блок-схеме не выполняют действий.(1.1.block)    
1.1.step   Шаги алгоритма - действия, решающие конкретную задачу. (1.1.alg)   
1.1.beg    Бегунок — это воображаемая точка, которая перемещается по алгоритму. (1.1.alg)    
1.1.mbeg   Бегунок всегда перемещается из Начала в Конец. (1.1.beg)   
1.1.full   Алгоритм и блок-схема (1.1.valg,1.1.prog,1.1.mbeg,1.1.step,1.1.obr,1.1.oval,1.1.blow,1.1.bfig,1.1.bread) 

###1.2 Целое число    
1.2.type   Тип данных определяет множество допустимых значений.
1.2.sexpr  Простым выражением называется совокупность значений, операций и скобок, которая возвращает значение определенного типа данных. (1.2.type)
1.2.plus   Операция сложения записывается как "+".
1.2.minus  Операция вычитания записывается как "-".
1.2.multy  Операция умножения записывается как "*".
1.2.cdiv  Операция целочисленного деления записывается как "//".
1.2.odiv  Операция получения остатка от деления записывается как "%".
1.2.carif  К арифметическим операциям для целочисленного типа относят сложение, вычитание, умножение, деление и получения остатка от деления.  (1.2.plus,1.2.minus,1.2.multy,1.2.cdiv,1.2.odiv)
1.2.cres   Целочисленные арифметические операции возвращают целое число. (1.2.carif)
1.2.intmn  Множество целых чисел включает в себя положительные и отрицательные целые числа. 
1.2.int    Целочисленный тип данных включает множество целых чисел.(1.2.type,1.2.intmn)
1.2.intv   Целочисленное выражение - выражение, составленные из целочисленного типа данных и целочисленных операций. (1.2.type,1.2.sexpr,1.2.carif,1.2.int)
1.2.full Целое число (1.2.intv, 1.2.cres)

##1.3. Вещественное число
1.3.floatmn  Множество вещественных чисел включает в себя рациональные числа.
1.3.floatp   Вещественные числа записываются через точку (1.3.floatmn)
1.3.div  Операция деления записывается как "/".
1.3.farif  К арифметическим операциям для вещественного типа относят сложение, вычитание, умножение, деление. (1.2.plus,1.2.minus,1.2.multy,1.3.div)
1.3.float    Вещественный тип данных включает множество вещественных чисел.(1.2.type,1.3.farif,1.3.floatmn)
1.3.floatret Арифметические операции для двух вещественных чисел возвращают вещественное число. (1.3.farif)     
1.3.floatv   Вещественное выражение - выражение, составленные из вещественного типа данных и арифметических операций для вещественного числа. (1.2.type,1.2.sexpr,1.3.floatret,1.3.float)
1.3.full Вещественное число (1.3.floatp,1.3.floatv)

##1.4. Строковый тип данных
1.4.string Строковый тип данных включает в себя символы алфавита.(1.2.type)
1.4.ostring Строковый тип данных обозначается кавычками. (1.4.string)
1.4.okstring Для обозначения могут использоваться как одинарные, так и двойные кавычки. (1.4.ostring)
1.4.instring Можно обрамлять один тип кавычек с помощью другого. (1.4.okstring) 
1.4.full Строковый тип данных (1.4.instring)

###1.5 Переменные и присваивание      
1.5.var  Переменная — это ячейка для временного хранения данных.     
1.5.id  Идентификатор - имя для переменной с ограниченным алфавитом.     
1.5.idf  Первым символом идентификатора должна быть латинская буква.(1.5.id)     
1.5.ids  Вторым и последующим символом идентификатора может быть латинская буква или цифра.(1.5.id)     
1.5.iddup  Идентификатор не должен повторять зарезервированные слова в языке.(1.5.id)    
1.5.idreg  Идентификатор чувствителен к регистру.(1.5.id)    
1.5.desc  Для объявления переменной используется идентификатор (1.5.id)    
1.5.pvar Операция присваивания копирует значение с правой стороны в переменную слева. (1.5.var)
1.5.pbvar Перед присваиванием переменная должна быть объявлена (1.5.desc,1.5.pvar)
1.5.opris Операция присваивания записывается с помощью символа "=". (1.5.pvar)
1.5.nvar Когда переменной присваивают новое значение, старое стирается.(1.5.pbvar)
1.5.vartype Тип данных переменной принимается такой же как присваиваемое значение. (1.5.pvar,1.2.type)
1.5.prio  Операция присваивания имеет низший приоритет.    
1.5.full  Переменные и присваивание (1.5.prio, 1.5.vartype, 1.5.nvar, 1.5.opris,1.5.idreg,1.5.iddup,1.5.ids,1.5.idf)


###1.6 Логический тип данных   
1.6.b Логическое значение - обладает двумя состояниями истина и ложь.
1.6.true Значение Истина, обозначается как True.
1.6.false Значение Ложь, обозначается как False.
1.6.lop Логические операции возвращают логическое значение (1.6.b). 
1.6.equ К операциям сравнения относят проверки на равенство, неравенство, больше, меньше, больше или равно, меньше или равно.   
1.6.and Логическое И обозначается как and.    
1.6.or Логическое ИЛИ обозначается как or.    
1.6.not Логическое НЕ обозначается как not.    
1.6.tbl Логические операции вычисляются согласно таблице истинности.
1.6.bop К логическим операциям относят "И", "ИЛИ", "НЕ"(1.6.and,1.6.or,1.6.not,1.6.tbl).
1.6.bool Логический тип данных - включает два логических значения и логические операторы и операции сравнения.(1.6.b,1.6.true,1.6.false,1.6.bop,1.6.equ)   
1.6.full  Логический тип данных (1.6.bool, 1.6.lop)
1.6.lov Логическое выражение - выражение, результатом вычисления которого является логическое значение. (1.2.sexpr,1.6.b)

###1.7 Оператор ветвления if   
1.7.inst Инструкция - это законченный фрагмент исполняемого кода, не возвращающий значения.    
1.7.posl Последовательность — однократное выполнение инструкций в том порядке, в котором они записаны в тексте программы. (1.7.inst)   
1.7.vetka Ветка - последовательность, выполняемая в особом порядке. (1.7.posl) 
1.7.ovet Основная ветка - последовательность, начнинающая в точке входа в программу и оканчивающаяся в одной или нескольких точках выхода. (1.7.vetka)
1.7.usl Условие - оператор, который проверет истинность логического выражения и выбирает ветку для выполнения. (1.6.lov, 1.7.vetka)
1.7.vvet Вложенная ветка - последовательность, начинающаяся после условия и оканчивающаяся точкой выхода из уровня. (1.7.vetka, 1.7.usl)     
1.7.vetv Оператор ветвления — организует выполнение одной из вложенных ветвей, в зависимости от заданного условия. (1.7.vvet)
1.7.if Для записи условия в программе используется форма if <УСЛОВИЕ>: (1.7.usl)
1.7.lvl Уровень вложенности отделяет основную ветку от вложенных. (1.7.ovet,1.7.vvet)
1.7.lvl2 В программе может быть долее одного уровня вложенности. (1.7.lvl)
1.7.otst Отступ уровня вложенности обозначается одним знаком табуляции или четырьмя пробелами. (1.7.lvl)
1.7.ifin Для обозначения вложенной ветки используется отступ уровня вложенности. (1.7.vetv,1.7.otst)
1.7.ifout Для обозначения точки выхода из условия используется возвращение к предыдущему уровню вложенности. (1.7.otst)
1.7.onev Условный оператор с одной ветвью обозначается: (1.7.if,1.7.ifin,1.7.ifout)
	```
	if <Условие>:
		Вложеннная инструкция 1    
		...
		Вложеннная инструкция N
	#точка выхода
	```
1.7.full Оператор ветвления if(1.7.onev,1.7.lvl2)

###1.8 Блоки ветвления else-elif
1.8.main Основное условие - условие, находящиеся после ключевого слова if.
1.8.alt  Альтернативное условие - логически инвертированное основное условие.(1.8.main)
1.8.else Альтернативное условие обозначается ключевым слововом else.
1.8.elif Дополнительное условие обозначается ключевым слововом elif.
1.8.twov Условный оператор с прямой и альтернативной ветвью обозначается: (1.8.main,1.8.alt,1.8.else)
	```
	if <Основное условие>:
		Вложеннная инструкция 1    
		...
		Вложеннная инструкция N
	else: #Альтернативное условие
	    Вложеннная инструкция 1    
		...
		Вложеннная инструкция N
	#точка выхода основного и альтернативного условий
	```
1.8.casc Каскадный условный оператор с прямой ветвью обозначается: (1.8.main,1.8.alt,1.8.elif)
	```
	if <Основное условие>:
		Вложеннная инструкция 1    
		...
		Вложеннная инструкция N
	elif <Допллнительное условие>:
	    Вложеннная инструкция 1    
		...
		Вложеннная инструкция N
	#точка выхода каскада
	```
1.8.casc2 Каскадный условный оператор с альтернативной ветвью обозначается: (1.8.main,1.8.alt,1.8.else,1.8.elif)
	```
	if <Основное условие>:
		Вложеннная инструкция 1    
		...
		Вложеннная инструкция N
	elif <Допллнительное условие>:
	    Вложеннная инструкция 1    
		...
		Вложеннная инструкция N
	else: #Альтернативное условие
	    Вложеннная инструкция 1    
		...
		Вложеннная инструкция N
	#точка выхода каскада
	```

###1.9 Вложенные операторы if
1.9.vlo Вложенный условный оператор: (1.8.main,1.7.ifin,1.7.ifout,1.7.otst)
	```
	if <Внешнее условие>:
		if <Вложенное условие>:
			Вложеннная инструкция 1    
			...
			Вложеннная инструкция N
		#точка выхода вложенного условия
	#точка выхода внешнего условия
	```
1.9.full Каскадный и вложенный if(1.7.onev,1.9.vlo,1.8.casc,1.8.casc2,1.8.twov)

###1.10 Цикл с параметром for
1.10.telo  Тело цикла содержит последовательность инструкций. (1.7.posl)
1.10.bpus  Блок проверки условия - при истинности логического выражения внутри него продолжается выполнение тела цикла. (1.6.lov,1.10.telo)    
1.10.cycl  Цикл состоит из блока проверки условия и тела цикла. (1.10.bpus,1.10.telo)    
1.10.pre  Цикл с предусловием проверяет блок условия до начала выполнения тела цикла. (1.10.cycl)   
1.10.parc  Параметрический цикл является циклом с предусловием и позволяет настроить число повторений в блоке условий. (1.10.pre)    
1.10.parm  Параметр цикла - специальная переменная которая меняется автоматически в блоке проверки условия. (1.10.bpus)    
1.10.mno   Параметр цикла принимает одно из заданного множества значений. (1.10.parm)     
1.10.mod  Модификация параметра цикла происходит автоматически после каждого прохождения, с учётом заданного множества значений. (1.10.parm,1.10.mno)
1.10.init  Инициализация параметра цикла выполняется первым значением из заданного множества. (1.10.parm,1.10.mno)
1.10.range  Заданное множество значений определяется функцией range. (1.10.mno)     
1.10.head  Для записи параметрического цикла используется конструкция "for <ПЕРЕМЕННАЯ> in range(<НАСТРОЙКИ>):" (1.10.parc)
1.10.ots  Тело цикла выделяется следующим уровнем вложенности. (1.7.otst)
1.10.okon  Окончание тела цикла определяется возвратом к предыдущему уровню вложенности. (1.7.otst)
1.10.vlo Циклы могут быть вложенными (1.10.cycl)
1.10.full Цикл с параметром for(1.10.okon,1.10.ots,1.10.head,1.10.range,1.10.init,1.10.mod,1.10.parm,1.10.vlo)

###1.11 Цикл while
1.11.while  Цикл while является циклом с предусловием. (1.10.pre)
1.11.bpus  Блок проверки условия располагается сразу после ключевого слова while (1.11.while,1.10.bpus)
1.11.form  Цикл while обозначается: (1.11.bpus, 1.10.ots,1.10.okon)
```
while <УСЛОВИЕ ИСТИННО>:
    Повторяемая инструкция 1
	...
	Повторяемая инструкция N
#конец цикла
```
1.11.break Для остановки работы цикла используется ключевое слово break.(1.10.cycl)    
1.11.continue Для запуска следующей итерации цикла, без прохождения всех инструкций, используется ключевое слово continue.(1.10.cycl)    
1.11.vlo Циклы, как и условия могут вкладываться одни в другие. (1.10.cycl,1.7.usl)     
1.11.full Цикл while(1.11.continue,1.11.break,1.11.form,1.11.vlo)

###1.13 Функции
1.13.fun Функция — последовательность, к которой можно обратиться из другого места программы.(1.7.posl)

1.13.vlo Можно вызывать одну функцию для передачи аргументов другой.
